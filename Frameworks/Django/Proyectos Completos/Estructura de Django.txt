------------------------------------------------------------------------------------------------------------------------------------
Django tiene la siguiente filosofía:
	--> Aplicación: aplicación web, compuesta de modelos determinados (tablas en las bases de datos, plantillas...)
	--> Proyecto: colección de configuraciones y aplicaciones diferentes, es decir, está compuesto por 1 o más aplicaciones totalmente diferentes

************************************************************************
*** NO SE PUEDE EJECUTAR UNA APLICACIÓN DJANGO SIN TENER UN PROYECTO ***
************************************************************************

Los pasos que sigo para crear todo son:

1.- Si no hemos instalado Django, lo hacemos: pip install django
2.- Para comenzar un proyecto, codifico en terminal: django-admin startproject nombre_de_mi_proyecto
3.- Entramos en la carpeta recien creada: cd nombre_de_mi_proyecto
4.- "Arranca, Carlos, por Diosssss!!!": python manage.py startapp aplicacion_nueva_a_crear
5.- Cada aplicación nueva se crea: python manage.py startapp siguiente_aplicacion_nueva_a_crear

Cuando ya tenemos 1 o más de una aplicaciones, la estructura nos quedaría

proyecto|

	|	     |		    |		|- __init__.py
	|	     |		    |-- admin.py    --> archivo de configuración de la aplicación compilada (aplicación que vemos realmente), la que gestiona el admin de Django
